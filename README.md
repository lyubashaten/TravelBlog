# TravelBlog

**TravelBlog** — веб-приложение для общения и обмена опытом среди путешественников.  
Платформа позволяет пользователям делиться историями о поездках, оставлять отзывы (комментарии) и просматривать посты других участников.

Проект выполнен в рамках учебного задания.

---

| **Фронтенд** | Vue 3 (Composition API), Vue Router, Pinia, Axios |
| **Сборка** | Vite |
| **Стили** | Sass (SCSS) |
| **Тестирование** | Jest, Vitest, Vue Test Utils, Testing Library |
| **Линтинг** | ESLint (Vue 3 recommended) |
| **Дополнительно** | Core-js, Babel, ts-jest, @types/node |

> Проект написан на **JavaScript (ES6+)**, инфраструктура готова для TypeScript.  
> Основные библиотеки: `vue@3`, `pinia@3`, `vue-router@4`, `axios@1`.

## 🚀 Инструкция по запуску

### Требования
- Node.js (версия 16 или выше)
- npm / yarn / pnpm

### Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/lyubashaten/TravelBlog.git
cd travel-blog

# 2. Установить зависимости
npm install

# 3. Запустить сервер для разработки
npm run dev

# 4. Сборка для продакшена
npm run build

# 5. Предпросмотр собранного приложения
npm run serve

# 6. Запуск unit-тестов
npm run test:unit   # Vitest
npm run test        # Jest


✅ Реализованный функционал (задачи)
Ниже перечислены все ключевые возможности, разработанные в рамках технического задания от учебной платформы.

1. Страницы авторизации и регистрации
Форма регистрации (email, пароль, подтверждение пароля) с валидацией.

POST-запрос к /api/register – создание нового пользователя.

Форма авторизации (email, пароль) с валидацией.

POST-запрос к /api/login – авторизация и сохранение сессии.

2. Главная страница со списком постов
GET-запрос к /api/posts – получение и отображение карточек постов (заголовок, краткое описание, страна, город).

Клик по карточке → переход на детальную страницу поста.

Для авторизованных пользователей отображается кнопка «Добавить моё путешествие» (неавторизованным скрыта).

3. Детальная страница поста с блоком комментариев
GET-запрос к /api/posts/{id} – полная информация: фото, заголовок (до 255), описание (до 2000), данные автора (ФИО, город, краткое описание).

Отображение всех комментариев (имя, дата, текст).

Для авторизованных – кнопка «Ваше впечатление об этом месте», открывающая форму создания комментария.

4. Форма создания нового поста
Поля: заголовок (до 255), описание (до 2000), фотография (JPEG/PNG), страна (до 255), город (до 255).

Все поля обязательны, клиентская валидация.

POST-запрос к /api/posts – создание поста.

5. Профиль пользователя
GET-запрос к /api/user – загрузка информации о пользователе.

Редактирование персональных данных (ФИО, город, фото, краткое описание) с валидацией (ФИО – обязательно).

Изменение пароля (PATCH /api/user/password).

Одновременная отправка PATCH (пароль) и POST (остальные данные) при нажатии «Сохранить».

Просмотр профиля по клику на иконку личного кабинета в хедере.

6. Форма создания комментария
Поля: ваше имя (до 255), комментарий (до 600) – оба обязательны.

POST-запрос к /api/posts/{id}/comments – добавление отзыва.

7. Навигация и интерфейс
Хедер с иконкой пользователя (для авторизованных) и кнопкой выхода.

GET-запрос к /api/logout – выход из аккаунта.

Разграничение доступа:

просмотр постов и деталей – для всех;

создание постов и комментариев – только для авторизованных.

8. Технические и организационные требования
Валидация данных на клиенте + обработка ошибок API (понятные сообщения).

Маршрутизация через vue-router (мультистраничность).

Управление состоянием через Pinia (хранилище пользователя, постов, комментариев).

Проект разделён на директории:

pages/ (или views/) – страницы приложения,

components/ – переиспользуемые UI-компоненты,

api/ – модули для работы с бекендом,

store/ – Pinia-сторы,

router/ – конфигурация маршрутов,

assets/ – стили, изображения.

Соблюдение макета (Pixel Perfect) и документации Swagger.
